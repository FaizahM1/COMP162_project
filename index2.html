Cybersecurity Intrusion Detection Dataset

link: https://www.kaggle.com/datasets/dnkumars/cybersecurity-intrusion-detection-dataset

each observation represents a single network session, its one row of data showing what happened during that specific session

such as like packet size, login attempts, encryption used, browser, and more, and whether a attack was detected

i picked this dataset because cybersecurity is the part of cs im most interested in, most of my classes next semester revolve around it

so i thought it would be nice learn a bit more about trends in the field

variables:

session_id: unique session identifier. qualitative, nominal

network_packet_size: size of packets. quantitative, continuous

protocol_type: type of network protocol. qualitative, nominal

login_attempts: number of login tries. quantitative, discrete

session_duration: how long the session lasted (in seconds). quantitative, continuous

encryption_used: type of encryption used. qualitative, nominal

ip_reputation_score: suspiciousness score (0 to 1). quantitative, continuous

attack_detected: 1 if attack happened, 0 if not. qualitative, binary (nominal)

failed_logins: number of failed login attempts. quantitative, discrete

browser_type: browser used (chrome, firefox, etc.). qualitative, nominal

unusual_time_access: 1 if session happened at weird time, 0 if not. qualitative, binary (nominal)

changes:

login attempts: only sessions w 10 or fewer login attempts were kept, so no extreme outliers

sessions exceeding 3600 seconds (1 hour) were excluded, as they could represent unusual activity

only sessions with 3 or fewer failed login attempts were included, higher values may indicate potential attacks or compromised

accounts, which will be addressed separately

lasso = Lasso(alpha=0.01)
lasso.fit(X_train_scaled, y_train)

y_pred = lasso.predict(X_test_scaled)

y_pred_binary = [1 if i >= 0.5 else 0 for i in y_pred]  # assuming 0.5 threshold for attack detection

rmse = mean_squared_error(y_test, y_pred_binary, squared=False)
